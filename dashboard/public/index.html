Agent System Overview Reference
Roco Agent System
Two agents are active. Four are planned for a future phase.
Active Agents
SELECTOR
Active
Role: Investor Scoring & Ranking
What it does: Evaluates every candidate investor against the deal brief. Scores each firm 0–100 across geography match, check size fit, sector alignment, investor type, and recent activity. Hard-rejects geography mismatches. Returns the ranked shortlist Roco uses to fill the 100-firm pipeline. Also reads agentic memory to apply learnings from previous cycles.
COMPOSER
Active
Role: Justification & Personalisation
What it does: Writes the research justification and personalisation seeds for each firm selected by SELECTOR. Produces the "why this firm fits this deal" text visible on each firm card, and the personalisation context used when generating outreach copy. Reads weekly intelligence to apply copy direction that has been working.
Planned Agents
IRIS
Coming Soon
Role: Reply Intelligence & Sentiment
What it does: Monitors all inbound replies across email and LinkedIn. Classifies sentiment (interested / not interested / needs more info / out of mandate). Drafts the follow-up response for Dom's approval when someone replies positively. Flags firms that have gone cold.
SCOUT
Coming Soon
Role: Batch Quality Review
What it does: Reviews each batch of 20 firms before it goes to Dom for approval. Flags any that look like a poor fit — wrong geography, check size mismatch, inactive fund, duplicate. Provides an overall batch rating (Strong / Acceptable / Weak) with reasoning.
SAGE
Coming Soon
Role: Weekly Intelligence & Learning
What it does: Runs every Monday at 9am. Analyses the past week's outreach performance — reply rates, acceptance rates, best sending times, which lists performed, which copy angles worked. Writes the weekly intelligence report and updates the intelligence digest that feeds back into SELECTOR and COMPOSER.
HERALD (future)
Coming Soon
Role: Outreach Execution
What it does: Handles the actual sending layer — LinkedIn invites, DMs, and emails via Unipile. Manages rate limits, timing, and delivery confirmation. Currently handled directly by the orchestrator.
